#### 会话管理命令

**列出所有会话**：
```bash
zellij ls
# 或
zellij list-sessions

- 显示所有活跃的会话和已退出的会话（EXITED 状态）
- Session 名称使用绿色加粗格式
- 默认按创建时间排序（最老的在前）
- 使用 `--reverse` 参数可以反转排序顺序
- 使用 `--short` 参数只显示会话名称
- 使用 `--no-formatting` 参数不添加颜色和格式（便于解析）

**启动新会话**：

```bash
zellij -s 会话名
# 或
zellij --session 会话名
```

- `-s` / `--session` 是全局参数，用于指定会话名称
- 如果会话不存在，会创建新会话
- 如果会话已存在，会连接到该会话

**连接到会话**：

```bash
zellij attach [会话名或前缀]
# 或
zellij a [会话名或前缀]
```

- 如果不指定会话名，会尝试连接最近的活跃会话：
  - 如果只有一个活跃会话，自动连接
  - 如果有多个活跃会话，会提示需要指定会话名
  - 已退出的会话（EXITED 状态）不会被连接
- 如果指定会话名或前缀：
  - 支持完整会话名或唯一前缀匹配
  - 如果前缀匹配多个会话，会列出所有匹配的会话并退出
  - 如果会话不存在，会报错（除非使用 `--create` 参数）

**杀死会话**：

```bash
zellij kill-session 会话名
# 或
zellij k 会话名
```

- 停止指定会话的运行
- 会话会变成 EXITED 状态（可以复活）

**删除会话**：

```bash
zellij delete-session 会话名 [--force]
# 或
zellij d 会话名 [--force]
```

- 删除已退出的会话（EXITED 状态）
- 如果会话正在运行，需要使用 `--force` 参数先杀死再删除

**杀死所有会话**：

```bash
zellij kill-all-sessions [--yes]
# 或
zellij ka [--yes]
```

- 停止所有运行中的会话
- 所有会话会变成 EXITED 状态（可以复活）
- 使用 `--yes` 参数可以跳过确认提示

**删除所有会话**：

```bash
zellij delete-all-sessions [--yes] [--force]
# 或
zellij da [--yes] [--force]
```

- 删除所有已退出的会话（EXITED 状态）
- 使用 `--force` 参数会先杀死所有运行中的会话，然后删除
- 使用 `--yes` 参数可以跳过确认提示

#### 命令说明

**kill vs delete**：

- `kill`：停止运行中的会话，会话变成 EXITED 状态，可以复活
- `delete`：清理已退出的会话（EXITED 状态），彻底删除，无法复活

**会话状态**：

- **活跃（Active）**：会话正在运行，有客户端连接
- **EXITED**：会话已退出，但可以复活（通过 `attach` 命令）

**attach 命令行为**：

- 不指定会话名时，只连接活跃会话，不连接 EXITED 状态的会话
- 如果只有一个活跃会话，自动连接
- 如果有多个活跃会话，需要指定会话名或前缀
